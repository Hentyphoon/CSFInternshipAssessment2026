The current issue I noticed is how router handles SQL queries directly along with data transofrmation, pagination and edge case. If the application was to scale this could create issues such as longer and more complex routes, difficulty in maintaining invidiual functionality and poor scalibilty as adding more functionality to already complex router code would be deterimental to the codebase. Thus, I would divide each functionality to layers, each handling http requests, buisness logic, database access, database connection seperately. This would increase readability of the code as each code now serves a more specific purpose and is more concise. In addition, since responsibility is divided upon different layers, it will be easier to locate the issue and fix it. Adding more functionality is also easier because developers can extend the appropriate layer without continuously increasing the complexity of route handlers.
